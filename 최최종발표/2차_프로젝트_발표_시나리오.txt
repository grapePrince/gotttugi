

그럼 다음은 2차 발표를 시작하겠습니다. 

2차로 제작한 페이지는 오뚜기 식품의 브랜딩 페이지입니다. 
오뚜기는 다양한 제품군을 보유한 식품 브랜드로
평소 자주 광고등에 보여 많은 사람들에게 친숙한 이미지를 가지고 있습니다. 

오뚜기는 사이트가 보유한 리소스가 많다는 점
제공하는 컨텐츠가 많다는 점 등 여러 장점을 가진 사이트였지만
크롬에서 깨져나오거나 모바일 페이지가 없는 등 아쉬운 부분이 있어 사이트를 리뉴얼하게 되었습니다. 

이번에도 빙그레 사이트나 청정원, 해태 등 다른 유명한 식품브랜드 사이트를 벤치마킹하여 보았고
여러 사이트를 분석한 결과 심미성과 사용성에 중점을 두고 리뉴얼하기로 하였습니다. 
 
그래서 리뉴얼 목표로는 
모던한 디자인을 지향하고
사이트맵을 깔끔하게 정리하고 
반응형 웹사이트를 제작하는 것을 목표로 잡았습니다. 

이렇게 목표를 세우고 난 뒤 스토리보드를 작성하여 기획을 하였고
스타일 가이드를 작성하여 디자인을 하였습니다. 
자세한 내용은 구현을 보여드릴 때 설명하도록 하겠습니다. 

현재 구현된 페이지는 메인, 제품목록, 제품상세, 공장견학 페이지입니다. 

개발 관련해서는 로고및 작은 이미지들을 스프라이트 이미지로 만들어 페이지 로딩 속도를 빠르게 해주었고
모바일에 대응하기 위해 2배수 이미지를 넣어주었습니다. 

개발을 진행할 때 npm 을 이용하여 자동으로 코드를 빌드해주도록 개발환경을 구축해 주었고요.

오뚜기 사이트에는 리소스들이 많이 있어서 동영상 같은 크기가 큰 리소스는 외부 호스팅을 이용해
서버 트래픽을 절약하였습니다. 

페이지들은 전부 웹표준을 통과하였고 
탭을 눌렀을 때 바로 본문 내용으로 이동할 수 있도록 바로가기를 만들어 주었습니다. 
역시 OG태그를 넣어주었습니다. 


여기까지 2분 30초 



그럼 다음은 기능 및 구현 내용을 설명하도록 하겠습니다.

먼저 사이트 전체적으로 타이틀은 전통적 느낌을 주는  noto serif 체를 사용하였고 본문은 noto sans 를 사용하였습니다. 
색깔은 오뚜기 포인트컬러인 노란색, 빨간색을 포인트 컬러로 잡아 활용하였습니다. 

오뚜기 상품을 홍보하는 것이 주 목적이기 때문에 
실제 구매로 이어질 수 있도록 오뚜기몰 링크를 좌측 상단에 배치하였고

우측 상단의 버튼들은 아이콘을 적절하게 넣어주어 공간을 작게 차지하면서도 
어떤 버튼인지 알려줄 수 있도록 하였습니다. 
마우스를 오버하면 아이콘이 왼쪽으로 밀리면서 글씨가 나와서 버튼 용도를 확실하게 알 수 있게하였고요.

언어선택의 경우 누르면 드롭다운 메뉴가 나타나고 
검색창은 포커스를 주면 하이라이트 컬러가 들어갑니다.

서브메뉴는 클릭하면 메뉴가 나오고 서브메뉴제목에 마우스를 오버하였을 때 이미지가 스르륵 나옵니다. 

사이트 전체적으로 반응형이 구현되어 있는데요.
헤더의 경우 태블릿 사이즈로 줄이면 검색창이 위로 올라가서 눌렀을 때 검색창이 나오게 되고 
더 줄이면 오뚜기몰 링크가 사라지고 양옆으로 메뉴가 갈라집니다. 

배너는 슬라이드 라이브러리로 구현 하였고요.
아래쪽에 보이는 버튼을 누르면 다음 단계로 자동으로 이동하며
위로가기 버튼을 누르면 위로 올라옵니다. 

회사소개는 뒤쪽에 광고 비디오를 넣어주었는데요. 
마우스를 스크롤해서 화면에서 나가면 비디오가 멈추고 다시 화면으로 들어오면 
비디오가 처음부터 재생되도록 구현하였습니다. 

신제품  소개 쪽은 이런 식으로 화면을 리사이즈 하면 아래로 쌓이게 되는데요. 
신제품 목록은 정사각형을 유지하면서 비율이 줄어들게 됩니다. 
이렇게 구현하기 위해서 내부적으로 padding-bottom 을 100%로 주어서 가로와 세로가 같아지는 기술을 사용하였습니다. 

오뚜기 소식들도 슬라이드로 되어있고 현재 슬라이드가 살짝 커지게 하였습니다. 
화면을 줄이면 이런식으로 슬라이드 너비도 도 리사이징 됩니다. 

인기상품 쪽에서는 상품을 광고해주기 위해 광고 동영상을 넣어주었는데요. 
동영상을 클릭하게 되면 화면의 너비에 맞추어 비디오가 가운데로 오도록 구현해주었습니다. 

레시피 쪽에는 각 레피시 순서 및 설명이 나오는데요. 
내부적으로 SVG를 이용하여 동그라미가 채워지는 애니메이션을 만들어 주었고 
마우스를 특정 단계에 오버하면 마우스가 나갔을 때에는 다음 단계부터 시작하도록 해주었습니다. 
하지만 모바일의 경우 마우스 오버가 없기 때문에 모바일에서는 모든 단계가 그냥 나타나도록 하였습니다. 

다음은 공장 체험 섹션인데요. 
스크롤을 내리면 각 요소들이 떠오르도록 구현하였습니다. 

아래의 미디어 속 오뚜기 섹션에서는 탭 메뉴를 만들어서 각 탭을 누를 때마다 컨텐츠가 보이도록 하였습니다. 

푸터는 심플하게 구성하였습니다. 

다음은 제품목록을 설명하도록 하겠습니다. 

서브 페이지의 경우 헤더 모양이 조금 바뀌게 되고 
브레드 스크럼이 들어가서 바로 링크를 눌러 이동할 수 있게 하였습니다.

제품목록 쪽은 스크롤을 내리면 목록이 더보기 방식으로 더 붙게 되어있고요. 
위쪽의 필터 버튼을 누르면 차종류인것, 차종류이면서 농/축/수산인것 그러면서 냉장/냉동인 것 
이렇게 여러 조건으로 필터링 해서 볼 수 있습니다. 
필터와 소팅 조건 역시 중복으로 사용할 수 있습니다. 

다음은 제품 상세 페이지인데요. 
스크롤을 내리면 숫자가 올라가게 구현하였고 
옆의 제품 사진은 클릭하였을 때 돌아가도록 만들어주었습니다. 
원산지 정보는 클릭했을 때만 노출됩니다. 

제품 상세쪽에는 표가 들어가 있는데요. 
이런 식으로 사이즈를 줄이면 반응형으로 가로와 세로가 바뀌면서 세로로 길게 변하도록 구현하였습니다. 

제품 상세 쪽 비디오 역시 클릭하면 큰 화면으로 볼 수 있습니다. 

아래쪽에는 다른 제품을 홍보하는 부분을 넣어주었습니다. 

마지막은 공장견학 페이지인데요. 
위쪽에 헤더 부분이 있어서 현재 위치가 하이라이트되도록 하였고요.

날짜 선택 부분은 달력을 참고로 볼 수 있는데 오늘 날짜 이후로 5일간의 날씨를 보여주도록 하였습니다. 
날씨 api 로는 openweather api 를 활용하였는데 이 api 가 시간별로 data 를 내려주고 있어서
각 data 를 돌면서 하루 전체로 봤을 때 최저/최고 기온을 찾아서 넣어주는 로직을 넣어주었습니다. 
아이콘의 경우 오후 3시의 아이콘을 사용합니다. 

이 공장견학 페이지 역시 반응형이 구현되어 있는데요. 
달력 부분은 참고용이기 때문에 모바일 버전에서는 아예 안보이도록 하였습니다
